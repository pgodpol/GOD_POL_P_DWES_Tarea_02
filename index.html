NOTAS PARA EVALUACIÓN DE LA TAREA Y EL SITIO WEB

Tarea realizada por Pedro Godoy Polaina, alumno de DAW en IES Trassierra, para la Ud.2 de DWES.

Hay 6 usuarios de base de datos registrados en la tabla 'usuario', cada uno con su rol asociado: profesor o alumno
Para comprobación de acceso se puede usar:

  • Rol de profesor(administrador): profesor, Contraseña: pppp
  • Rol de alumno(usuario): alumno, Contraseña: aaaa
Hay tres alumnos de los que se puede extraer sus datos de examen: carlos, david, pedro.
Cada rol de usuario tiene asociados un uso distinto de la página.

EN RELACIÓN A LA BASE DE DATOS

Los parámetros para acceso a la base de datos son los siguientes:

  • Servidor = "localhost"
  • Usuario = "root"
  • Password = ""
  • Nombre de la BD = "ud2_test"

Para la programación de la base de datos, basándome en el texto de la tarea he realizado un
diagrama E/R con todas las entidades y relaciones que he considerado conveniente incluir en la BD.
Posteriormente he pasado a tablas el diagrama, comrpobando gráficamente que el modelo es funcional.
El ultimo término he programado en lenguaje MySQL para MariaDB todas las tablas y restricciones de la
Base de datos quedando dispuesta y sin errores para la creación con el gestor phpMyAdmin.

He adjuntado la base de datos tal y como se pide en formato .sql exportado desde phpMyAdmin,
aunque también he aportado en formato .txt la base de datos tal y como yo la programé y volqué
inicialmente.

CONTENIDO DE LA TAREA

  • Página de incio desde donde se puede navegar e interactuar con la aplicación.
  • Incluye conexión con una base de datos.
  • Inicio de sesión tanto para el rol de usuario como para el rol de administrador.
  • Incluye una página de registro o alta como usuario nuevo.
  • Incluye sesión como alumno (usuario), y administrador (profesor).
  • Una vez iniciada la sesión cada rol de usuario tiene un tipo de servicio distinto.
  • Se pueden generar distintos informes recogiendo la información de una base de datos (MariaDB).
  • Es posible generar un examen de 10 preguntas con la capacidad de enviar las respuestas a la base de datos
  • Es posible generar un informe exportado a tabla con las respuestas dadas por un alumno concreto.
  • Es posible generar un informe exportado a tabla con las notas alcanzadas por un alumno concreto.
  • Cerrar sesión tanto como usuario como administrador.
  • Se realizan comprobaciones de usuario por seguridad para servir los datos.
